# 網站大改版變革 v1


## 關鍵內容

- 新增 正常模式 與 隨機模式 的明確規則：
    - 正常模式必須選擇早餐、午餐或晚餐，並只從該餐別有效卡牌等機率抽取。
    - 隨機模式不需選餐別，直接從早餐、午餐、晚餐全部有效卡牌的大池等機率抽取。
    - 空卡池、資料封鎖、驗證失敗或重複送出不得產生成功結果。
- 新增卡牌機率統計的業務定義：
    - 總抽取次數 = 正常模式與隨機模式的全部成功抽卡次數。
    - 單一卡牌歷史機率 = 該卡成功抽中次數 / 總成功抽卡次數。
    - 只統計成功抽卡；驗證失敗、資料封鎖、空卡池、未投幣或重複提交不計入。
    - 已刪除卡牌保留歷史列並標示已刪除，避免總數與歷史機率失真。
    - 卡牌改名或翻譯更新不切分統計，歷史紀錄綁定不可變卡牌 ID。
- 新增統計表顯示規格：
    - 首頁顯示本網站總成功抽取次數。
    - 每列顯示卡牌名稱、餐別、抽中次數、歷史機率、卡牌狀態。
    - 無抽卡紀錄時顯示可理解的空狀態，不顯示誤導性的機率。
    - 語系切換只影響顯示文字，不改變統計分母、抽中次數或卡牌身分。
- 補強 UX 業務邏輯邊界：
    - 華麗賭場/皇室風格只能提升呈現，不得改變抽卡機率。
    - 鼓勵文案必須基於真實紀錄或泛用鼓勵，不得宣稱「下一次機率更高」等不實暗示。
    - 不新增付費、下注、點數、賭金、稀有度或好牌等級規則。
    - 必須支援 reduced motion，且動畫時間不得影響抽卡結果。

## 公開介面與資料概念

- 規格中定義未來可用的概念型別：DrawMode、DrawHistoryRecord、CardDrawStatistic。
- 首頁抽卡表單新增模式選擇；mealType 只在正常模式必填。
- 統計表是使用者可見公開介面；不新增外部 JSON API。
- 歷史紀錄應具備原子性：抽卡結果與統計紀錄必須一起成立；紀錄失敗時不得宣告抽卡成功。

## 測試與驗收情境

- 正常模式未選餐別會被拒絕，且不增加統計。
- 正常模式只會抽出所選餐別卡牌。
- 隨機模式不需餐別，且可抽出任一餐別有效卡牌。
- 每次成功抽卡只新增一筆歷史紀錄。
- 總抽取次數與每張卡機率依全部成功抽卡正確計算。
- 已刪除卡牌仍出現在統計表並標示已刪除。
- 卡牌改名後歷史統計仍歸屬同一張卡牌 ID。
- 語系切換、動畫、顯示排序與 repeated click 不影響抽卡結果或統計。
- 資料封鎖、空卡池、寫入失敗時不產生成功抽卡紀錄。

## 假設

- 統計口徑採用你選定的「全部成功抽卡」。
- 刪除卡牌後採用你選定的「保留歷史列」。
