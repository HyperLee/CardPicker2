# UI 契約: 餐點抽卡網站2

## 範圍

本功能不暴露外部 JSON API。公開介面是 Razor Pages 的頁面、查詢字串、表單欄位、表單 handler、狀態碼、使用者可見訊息與前端互動狀態。所有狀態變更表單都必須包含 ASP.NET Core Anti-Forgery token。

## 全域規則

- 所有使用者可見文案使用繁體中文。
- 所有頁面必須使用 shared layout、Bootstrap 5 元件與 `wwwroot/css/site.css` 補充樣式。
- 所有表單錯誤必須同時呈現在欄位旁與 validation summary 或等效可及位置。
- 所有互動按鈕必須有可辨識文字；老虎機拉桿等裝飾不得成為唯一操作線索。
- 所有頁面在卡牌庫 blocking recovery 狀態時都必須顯示阻斷錯誤，並停用新增、編輯、刪除、抽卡等狀態變更操作。
- 錯誤訊息不得包含 stack trace、完整檔案內容、秘密值、連線字串或金鑰。

## GET `/` 首頁抽卡

**Page**: `Pages/Index.cshtml`

**目的**: 顯示賭場老虎機式餐點抽卡入口。

**輸入**:

| 來源 | 名稱 | 型別 | 必填 | 規則 |
|------|------|------|------|------|
| query | `mealType` | `Breakfast` / `Lunch` / `Dinner` | 否 | 提供時預先選取餐別；非法值顯示驗證錯誤並不啟動抽卡。 |

**必須呈現**:

- 餐別選擇控制，包含早餐、午餐、晚餐。
- 投幣或等效確認控制。
- 拉桿或開始抽卡控制。
- 老虎機 reel/slot 視覺區。
- 抽卡狀態區：未選餐別、已投幣、轉動中、結果揭示、錯誤。
- 結果揭示時顯示餐點名稱、餐別與完整描述。

**空狀態**:

- 若所選餐別沒有卡牌，顯示「這個餐別目前沒有可抽取的餐點卡牌。」
- 不得顯示假結果或上一筆結果作為替代。

## POST `/?handler=Draw` 執行抽卡

**PageModel handler**: `IndexModel.OnPostDrawAsync`

**目的**: 驗證餐別與卡池後產生單次等機率抽卡結果。

**表單欄位**:

| 名稱 | 型別 | 必填 | 規則 |
|------|------|------|------|
| `MealType` | `Breakfast` / `Lunch` / `Dinner` | 是 | 未提供或非法值時拒絕抽卡。 |
| `CoinInserted` | `bool` | 是 | 必須為 `true`；否則顯示「請先投幣再拉桿。」 |

**成功回應**:

- HTTP 200，回傳首頁 HTML。
- 頁面狀態為 `Revealed`。
- 顯示抽中卡牌的 `Name`、`MealType` 繁中顯示名稱與 `Description`。
- 記錄結構化 Information log：餐別、卡牌 ID、卡池數量。

**失敗回應**:

| 條件 | 回應 |
|------|------|
| 未選餐別 | HTTP 200，首頁顯示「請先選擇早餐、午餐或晚餐。」 |
| 未投幣 | HTTP 200，首頁顯示「請先投幣再拉桿。」 |
| 所選餐別無卡牌 | HTTP 200，首頁顯示「這個餐別目前沒有可抽取的餐點卡牌。」 |
| 卡牌庫 blocking recovery | HTTP 503 或 HTTP 200 阻斷頁面狀態，顯示復原錯誤並停用操作。 |
| Anti-Forgery 驗證失敗 | 使用 ASP.NET Core 預設 400 行為。 |

**前端互動規則**:

- 送出後立即停用開始/拉桿按鈕，避免重複提交。
- `prefers-reduced-motion: reduce` 時不播放連續轉動動畫，只顯示短暫靜態揭示。
- 動畫不得自行產生或覆寫抽卡結果。

## GET `/Cards`

**Page**: `Pages/Cards/Index.cshtml`

**目的**: 瀏覽與搜尋卡牌庫。

**查詢字串**:

| 名稱 | 型別 | 必填 | 規則 |
|------|------|------|------|
| `keyword` | string | 否 | trim 後空字串視為未提供；大小寫不敏感部分比對 `Name`。 |
| `mealType` | `Breakfast` / `Lunch` / `Dinner` | 否 | 提供時只顯示該餐別。 |

**成功回應**:

- HTTP 200。
- 顯示搜尋表單。
- 列表預設顯示每張卡牌的餐點名稱與餐別。
- 每筆提供查看詳情入口；編輯與刪除入口僅在卡牌管理故事完成後提供，避免瀏覽搜尋故事獨立交付時出現無法使用的管理操作。

**無結果回應**:

- HTTP 200。
- 顯示「查無符合條件的餐點卡牌。」
- 保留搜尋表單與清除條件入口。

## GET `/Cards/{id}`

**Page**: `Pages/Cards/Details.cshtml`，`@page "{id:guid}"`

**目的**: 顯示單一卡牌完整描述。

**路由參數**:

| 名稱 | 型別 | 必填 | 規則 |
|------|------|------|------|
| `id` | `Guid` | 是 | 必須對應仍存在的卡牌。 |

**成功回應**:

- HTTP 200。
- 顯示餐點名稱、餐別與完整描述。

**失敗回應**:

- 找不到卡牌時 HTTP 404，顯示繁體中文找不到訊息。

## GET `/Cards/Create`

**Page**: `Pages/Cards/Create.cshtml`

**目的**: 顯示新增卡牌表單。

**必須呈現欄位**:

- 餐點名稱
- 餐別
- 描述

**成功回應**:

- HTTP 200。
- 表單包含 Anti-Forgery token。

## POST `/Cards/Create`

**PageModel handler**: `CreateModel.OnPostAsync`

**表單欄位**:

| 名稱 | 型別 | 必填 | 規則 |
|------|------|------|------|
| `Input.Name` | string | 是 | trim 後不得為空。 |
| `Input.MealType` | enum | 是 | 必須是支援餐別。 |
| `Input.Description` | string | 是 | trim 後不得為空。 |

**成功回應**:

- 建立成功後 redirect 至新卡牌詳情頁或卡牌列表頁。
- 顯示成功訊息「已新增餐點卡牌。」
- 新卡牌可立即被瀏覽、搜尋與抽卡使用。

**失敗回應**:

| 條件 | 回應 |
|------|------|
| 必填欄位缺失 | HTTP 200 回表單，欄位旁顯示必填錯誤。 |
| 餐別非法 | HTTP 200 回表單，顯示餐別錯誤。 |
| 完全重複卡牌 | HTTP 200 回表單，顯示「已有相同餐點名稱、餐別與描述的卡牌。」 |
| 寫入失敗 | HTTP 200 回表單，顯示「卡牌暫時無法儲存，請稍後再試。」並記錄 Error log。 |

## GET `/Cards/Edit/{id}`

**Page**: `Pages/Cards/Edit.cshtml`，`@page "{id:guid}"`

**目的**: 顯示編輯既有卡牌表單。

**成功回應**:

- HTTP 200。
- 表單預填目前卡牌的餐點名稱、餐別與描述。
- `Id` 只作為路由與隱藏欄位輔助，不可讓使用者改變卡牌 ID。

**失敗回應**:

- 找不到卡牌時 HTTP 404，顯示繁體中文找不到訊息。

## POST `/Cards/Edit/{id}`

**PageModel handler**: `EditModel.OnPostAsync`

**表單欄位**: 同新增卡牌，另含不可變 `Id`。

**成功回應**:

- 更新成功後 redirect 至卡牌詳情頁或列表頁。
- 顯示成功訊息「已更新餐點卡牌。」
- 更新內容立即反映在後續瀏覽、搜尋與抽卡流程。

**失敗回應**:

| 條件 | 回應 |
|------|------|
| 找不到卡牌 | HTTP 404。 |
| 必填欄位缺失或餐別非法 | HTTP 200 回表單並顯示欄位錯誤。 |
| 編輯後會與另一張卡牌完全重複 | HTTP 200 回表單，顯示「已有相同餐點名稱、餐別與描述的卡牌。」 |
| 寫入失敗 | HTTP 200 回表單，保留原卡牌內容並顯示儲存失敗訊息。 |

## POST `/Cards/{id}?handler=Delete`

**PageModel handler**: `DetailsModel.OnPostDeleteAsync`，由 `Pages/Cards/Details.cshtml` 的 `@page "{id:guid}"` 表單送出。

**目的**: 在使用者確認後永久刪除卡牌。

**表單欄位**:

| 名稱 | 型別 | 必填 | 規則 |
|------|------|------|------|
| `ConfirmDelete` | bool | 是 | 必須為 `true`；否則不得刪除。 |

**成功回應**:

- 刪除成功後 redirect 至 `/Cards`。
- 顯示成功訊息「已刪除餐點卡牌。」
- 被刪除卡牌不得再出現在列表、搜尋、詳情或抽卡結果。

**失敗回應**:

| 條件 | 回應 |
|------|------|
| 未確認刪除 | HTTP 200，顯示「請先確認刪除意圖。」 |
| 找不到卡牌 | HTTP 404。 |
| 寫入失敗 | HTTP 200，保留原卡牌並顯示刪除失敗訊息。 |

## 安全標頭契約

正式環境回應必須包含：

- `Strict-Transport-Security`，由 `UseHsts()` 提供。
- `Content-Security-Policy`，至少限制 `default-src 'self'`，script/style 來源僅允許本機靜態資源與必要 inline nonce/hash 策略。
- 不得加入會暴露內部實作或秘密值的診斷 header。

## 可及性與 responsive 契約

- 餐別選擇、投幣、拉桿、搜尋、建立、編輯、刪除確認都必須可用鍵盤操作。
- 抽卡進行中狀態必須有文字回饋，不能只靠動畫表示。
- 表單錯誤應可被螢幕閱讀器讀取。
- 行動寬度下按鈕、卡牌、老虎機視覺區與文字不得互相重疊或溢出。
