## 使用者情境與測試 *(mandatory)*

### 使用者故事 1 - 在首頁切換整站語系 (Priority: P1)

使用者在首頁可以在繁體中文與英文之間切換語系。切換後，首頁上的導覽、按鈕、功能名稱、狀態文字、餐別名稱與目前可見的餐點資訊都必須立即以選定語系呈現，讓不熟悉繁體中文的使用者也能理解並操作餐點抽卡網站。

**優先級理由**: 語系切換入口與首頁即時回饋是本功能的核心價值；若首頁不能清楚切換，使用者無法信任其他頁面已套用語系。

**獨立測試**: 可在首頁預設繁體中文狀態下切換至英文，再切回繁體中文，確認所有首頁可見文案、按鈕、功能名稱、餐別標示與目前可見餐點資訊都跟著改變，且抽卡狀態、搜尋條件或未送出輸入不被清除。

**驗收情境**:

1. **Given** 使用者第一次進入首頁且尚未選擇語系，**When** 使用者選擇英文，**Then** 首頁所有可見介面文字、功能名稱、按鈕文字、餐別名稱與目前可見餐點資訊都以英文呈現。
2. **Given** 使用者目前在英文語系首頁，**When** 使用者切回繁體中文，**Then** 首頁所有可見介面文字與餐點資訊都回到繁體中文。
3. **Given** 使用者在首頁已有可見抽卡結果或表單輸入，**When** 使用者切換語系，**Then** 系統只切換可見語言，不得清除結果、輸入值或目前操作狀態。

---

### 使用者故事 2 - 以選定語系使用抽卡與卡牌庫 (Priority: P2)

使用者選定語系後，前往卡牌庫、卡牌詳情、建立、編輯、刪除確認、隱私權與錯誤頁等站內頁面時，必須看到一致語系的導覽、功能名稱、表單、驗證訊息、餐別標示、餐點名稱與餐點描述。抽卡揭示結果也必須使用目前語系顯示。

**優先級理由**: 只切換首頁不足以支援英文使用者完成核心任務；餐點抽卡、瀏覽、搜尋與卡牌管理都必須能用同一語系完成。

**獨立測試**: 可在首頁選擇英文後依序瀏覽主要站內頁面、執行搜尋、查看卡牌詳情並完成一次抽卡，確認可見文案與餐點內容皆為英文；再切回繁體中文重複相同流程。

**驗收情境**:

1. **Given** 使用者已選擇英文，**When** 使用者前往卡牌列表或卡牌詳情，**Then** 導覽、功能名稱、按鈕、搜尋欄位、餐別標示、餐點名稱與餐點描述都以英文呈現。
2. **Given** 使用者已選擇英文且卡牌庫中存在英文餐點名稱，**When** 使用者以英文關鍵字搜尋，**Then** 搜尋結果只顯示符合目前語系可見餐點名稱的卡牌，且無結果訊息以英文呈現。
3. **Given** 使用者已選擇任一語系並完成抽卡，**When** 系統揭示結果，**Then** 抽出的卡牌身分與餐別必須維持正確，且餐點名稱、餐別與描述以目前語系呈現。
4. **Given** 使用者已選擇英文，**When** 表單驗證失敗或系統顯示確認、成功、失敗、復原訊息，**Then** 可見訊息必須以英文呈現且保持可操作。

---

### 使用者故事 3 - 維護雙語餐點內容與保留偏好 (Priority: P3)

使用者新增或編輯餐點卡牌時，可以維護繁體中文與英文的餐點名稱與描述，確保未來在任一語系下瀏覽、搜尋與抽卡都能顯示完整內容。使用者選定語系後，在同一瀏覽器與裝置再次造訪網站時，系統會沿用最近一次有效語系。

**優先級理由**: 餐點名稱是使用者明確要求可切換的內容；若卡牌資料無法維護雙語內容，英文介面會在核心結果上失效。偏好保留則避免每次造訪都重新切換。

**獨立測試**: 可新增一張含繁中與英文名稱/描述的卡牌，在兩種語系下搜尋、查看詳情與抽卡，確認同一張卡牌以對應語系顯示；重新開啟網站後確認語系偏好仍生效。

**驗收情境**:

1. **Given** 使用者正在新增或編輯卡牌，**When** 使用者提供繁體中文與英文餐點名稱及描述後送出，**Then** 該卡牌在兩種語系下都能被瀏覽、搜尋、查看詳情與抽卡結果顯示。
2. **Given** 使用者新增或編輯卡牌但缺少任一必要語系的餐點名稱或描述，**When** 使用者送出，**Then** 系統必須拒絕儲存並以目前語系指出缺少的內容。
3. **Given** 使用者已在同一瀏覽器與裝置選擇英文，**When** 使用者稍後重新開啟網站，**Then** 網站必須自動使用英文；若無有效偏好，則預設使用繁體中文。

### 邊界情況

- 使用者尚未選擇語系時，系統預設使用繁體中文。
- 儲存或讀取到不支援的語系偏好 cookie 時，系統必須回到繁體中文，且不得中斷抽卡、瀏覽、搜尋或卡牌管理。
- 使用者快速連續切換語系時，最後一次有效選擇必須成為目前語系，且頁面不得混用兩種語言造成無法理解的狀態。
- 使用者在表單輸入、搜尋結果、驗證錯誤、刪除確認或抽卡結果可見時切換語系，系統不得清除未送出輸入、搜尋條件、驗證訊息、確認狀態或已揭示結果。
- 卡牌缺少英文內容時，英文語系必須顯示繁體中文名稱與描述作為 fallback，不得顯示空白餐點名稱或描述，並提供清楚、可操作的補齊英文翻譯提示。
- 使用者在英文語系搜尋中文關鍵字，或在繁體中文語系搜尋英文關鍵字時，系統必須依目前語系的可見餐點名稱套用搜尋，並以目前語系顯示無結果或結果清單。
- 語系切換不得改變餐點卡牌 ID、餐別、抽卡機率、已刪除卡牌狀態、搜尋條件或任何餐點決策規則。
- 語系切換、翻譯缺漏提示與錯誤訊息不得揭露秘密值、連線字串、系統提示、堆疊追蹤或完整內部資料內容。

## 需求 *(mandatory)*

### 功能需求

- **FR-001**: 系統必須在 shared layout 中提供語系切換入口，使首頁與所有主要站內頁面都能切換語系，且支援語系僅包含繁體中文與英文。
- **FR-002**: 系統必須清楚顯示目前選取語系。
- **FR-003**: 使用者未選擇語系或語系偏好無效時，系統必須預設使用繁體中文。
- **FR-004**: 使用者在首頁切換語系後，系統必須立即以選定語系更新首頁所有可見使用者介面文字與目前可見餐點資訊。
- **FR-005**: 首頁選定語系必須套用到所有主要站內頁面，而不是只套用在首頁。
- **FR-006**: 首頁以外頁面必須使用目前語系顯示內容，且語系切換入口必須在首頁、卡牌庫、卡牌詳情、卡牌建立、卡牌編輯、刪除確認、隱私權頁與錯誤頁等主要頁面保持可見且可操作。
- **FR-007**: 系統必須以 ASP.NET Core 可於伺服器端讀取的語系偏好 cookie，在同一瀏覽器與裝置上保留使用者最近一次有效語系，讓後續 Razor Pages render、表單驗證訊息與餐點內容自動沿用。
- **FR-008**: 系統必須拒絕或忽略繁體中文與英文以外的語系 cookie 值，並回到繁體中文安全預設。
- **FR-009**: 所有使用者可見導覽、按鈕、連結、功能名稱、區塊標題、欄位標籤、提示文字、狀態文字、成功訊息、錯誤訊息、驗證訊息、確認訊息與復原訊息都必須能依目前語系呈現。
- **FR-010**: 早餐、午餐、晚餐等餐別顯示名稱必須能依目前語系呈現。
- **FR-011**: 卡牌列表、卡牌詳情、卡牌表單、刪除確認、搜尋結果與抽卡結果中所有可見餐點名稱與描述都必須能依目前語系呈現。
- **FR-012**: 搜尋關鍵字必須依目前語系的可見餐點名稱進行比對；搜尋結果與無結果訊息必須以目前語系呈現。
- **FR-013**: 抽卡結果必須維持原本選定卡牌與餐別，只改變可見語言；語系不得影響抽卡機率或結果身分。
- **FR-014**: 新增與編輯餐點卡牌時，系統必須允許使用者維護繁體中文與英文的餐點名稱與描述。
- **FR-015**: 新增或編輯後的卡牌若缺少任一必要語系的餐點名稱或描述，系統必須拒絕儲存並以目前語系顯示可操作錯誤。
- **FR-016**: 系統必須防止同一餐別中出現任一語系的正規化餐點名稱與描述都相同的餐點卡牌；只要繁體中文或英文任一語系的 `Name.Trim()` 與 `Description.Trim()` 以不區分大小寫方式比對後，同餐別已有相同組合，新增或編輯就必須被拒絕並以目前語系說明原因。
- **FR-017**: 切換語系不得清除或重新計算目前可見的抽卡結果、搜尋條件、未送出表單輸入、驗證訊息或刪除確認狀態。
- **FR-018**: 若既有卡牌缺少英文內容，英文語系必須顯示該卡牌的繁體中文名稱與描述作為 fallback，並提供明確補齊英文翻譯的操作提示；在英文內容補齊前，搜尋必須以目前可見的 fallback 名稱進行比對。
- **FR-019**: 語系偏好暫時無法保留時，系統不得阻斷抽卡、瀏覽、搜尋或卡牌管理；目前頁面仍應盡可能套用使用者剛選擇的語系。
- **FR-020**: 系統必須確保任何使用者可見訊息在目前語系下都不包含秘密值、連線字串、系統提示、堆疊追蹤或完整內部資料內容。

### 資料完整性需求 *(include if feature involves cards, selection, or persisted data)*

- **DI-001**: 同一張餐點卡牌在不同語系下必須代表同一個不可變卡牌身分；翻譯不得建立另一張抽卡用卡牌。
- **DI-002**: 新增或編輯後的餐點卡牌必須具備繁體中文與英文的非空餐點名稱與描述。
- **DI-003**: 語系切換不得改變餐點卡牌、餐別、抽卡結果、搜尋條件、未送出表單資料或刪除狀態。
- **DI-004**: 抽卡等機率規則必須與語系無關；同一餐別中的每張有效卡牌在任一語系下都維持相同被抽中機率。
- **DI-005**: 重複卡牌判斷必須涵蓋兩種語系的可見餐點名稱與描述；只要同一餐別中有另一張卡牌在繁體中文或英文任一語系形成相同的正規化名稱與描述組合，就必須視為重複並完整拒絕變更。
- **DI-006**: 缺漏翻譯不得造成空白名稱、空白描述、錯誤卡牌身分或不可追蹤的資料狀態；英文內容缺漏期間必須以原繁體中文內容作為顯示 fallback，並保留翻譯缺漏狀態供使用者補齊。

### 非功能需求

- **NFR-001**: 治理文件必須更新為允許產品執行時介面依使用者選擇呈現繁體中文或英文；本規格與開發文件仍維持繁體中文撰寫。
- **NFR-002**: 語系切換後，主要頁面必須在桌面、平板與行動尺寸下保持可操作，文字、按鈕、卡牌、表單與抽卡元素不得重疊或水平溢出。
- **NFR-003**: 語系切換入口、表單、搜尋、抽卡與卡牌管理流程必須符合 WCAG 2.1 AA 可及性目標，包含鍵盤操作、可見焦點與不只依賴顏色傳達狀態。
- **NFR-004**: 語系切換不得讓主要抽卡、瀏覽、搜尋、建立、編輯與刪除流程超出既有效能預算；使用者切換後必須快速看到一致語系。
- **NFR-005**: 所有狀態變更提交都必須驗證輸入，並具備防止未授權或非預期提交的保護。
- **NFR-006**: 關鍵語系切換、偏好套用、翻譯缺漏與驗證失敗情境必須可被診斷，但診斷資訊不得包含秘密值、系統提示、完整內部資料內容或未清理的使用者輸入。

### 關鍵實體 *(include if feature involves data)*

- **語系選項**: 使用者可選擇的顯示語言，包含繁體中文與英文；繁體中文是預設語系。
- **使用者語系偏好**: 同一瀏覽器與裝置上最近一次有效語系選擇，儲存在 ASP.NET Core 可讀取的 cookie 中，用於後續造訪時自動沿用。
- **本地化介面文字**: 導覽、按鈕、功能名稱、欄位標籤、狀態、驗證、確認、成功、錯誤與復原訊息在各語系下的可見文字集合。
- **本地化餐點內容**: 同一張餐點卡牌在繁體中文與英文下的餐點名稱與描述，必須維持同一卡牌身分與餐別。
- **語系套用範圍**: 所有目前可由使用者直接瀏覽與操作的主要站內頁面，包括首頁、卡牌庫、卡牌詳情、卡牌建立、卡牌編輯、刪除確認、隱私權頁與錯誤頁；語系切換入口由 shared layout 提供。

## 成功標準 *(mandatory)*

### 可量測結果

- **SC-001**: 95% 的使用者能在首頁或任一主要站內頁面 10 秒內找到語系切換入口並完成繁體中文與英文之間的切換。
- **SC-002**: 使用者在首頁切換語系後，首頁與後續開啟的主要站內頁面在 1 秒內呈現一致語系。
- **SC-003**: 100% 的主要站內頁面在繁體中文與英文兩種語系下，都不出現未翻譯的導覽、按鈕、功能名稱、餐別標示、表單標籤、狀態訊息、驗證訊息、確認訊息或錯誤訊息。
- **SC-004**: 100% 的預載餐點卡牌與本功能上線後新增或編輯的卡牌，在兩種語系下都顯示非空餐點名稱與描述。
- **SC-005**: 100% 的抽卡結果在任一語系下都顯示目前語系的餐點名稱、餐別與描述，且不出現不屬於所選餐別、已刪除或空白名稱的結果。
- **SC-006**: 100% 的語系切換情境不清除未送出表單輸入、搜尋條件、驗證訊息、刪除確認狀態或已揭示抽卡結果。
- **SC-007**: 在同一瀏覽器與裝置可保留偏好的情境中，100% 的回訪會自動使用使用者最近一次有效語系；無有效偏好時 100% 預設為繁體中文。
- **SC-008**: 主要頁面在 390x844、768x1024 與 1366x768 檢查尺寸下，兩種語系都 100% 不出現文字、按鈕、表單或卡牌內容重疊與水平溢出。
- **SC-009**: 使用者以目前語系搜尋餐點名稱時，100% 的結果清單與無結果訊息都符合目前語系，且不顯示空白餐點名稱。
- **SC-010**: 100% 缺少英文內容的既有卡牌在英文語系下都顯示繁體中文 fallback 名稱與描述、補齊英文翻譯提示，且不出現空白餐點名稱或描述。

## 假設

- 繁體中文是預設語系，也是無偏好、偏好 cookie 無效或偏好 cookie 不可用時的安全預設。
- 語系切換入口由 shared layout 提供，因此首頁與其他主要頁面都必須可直接切換語系。
- 英文餐點名稱與描述由使用者或預載資料維護；本功能不假設自動翻譯外部服務。
- 預載餐點卡牌需要提供繁體中文與英文名稱及描述，才能完整支援兩種語系的抽卡與瀏覽體驗。
- 既有單語卡牌若缺少英文內容，仍必須保留原有卡牌與抽卡資料完整性；英文語系下必須顯示繁體中文 fallback 內容，且必須引導使用者補齊英文內容。
- 語系模式與網站主題模式是彼此獨立的使用者偏好；兩者都可能出現在首頁，且任一偏好切換不得清除另一個偏好或目前頁面狀態。
